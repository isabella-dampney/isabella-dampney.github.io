Reflect on how confident you feel using the problem solving techniques and process:

Pseudocode

Pseudocode is notes to describe what your code is doing or what you want your code to do. They are short, prosaic, and descriptive. Pseudocode should be helpful, and not clutter your working space. Pseudocode is intended for humans to read and not for the computer. You can write pseudocode by “commenting out” the section with a “//”.

Using pseudocode in problem solving means that you work through and plan out what it is that you want your code to achieve. In writing it out you are already working through your ideas and problem solving how the code will take shape.

You can approach your pseudocode as written language comments, or you can write it following the syntax of the language you are working with. There is no guidebook on pseudocode, but as we have learnt with readability the important concept here is that it is consistent and logical for others to follow.

Rubber Ducky Method

I have read about this method before, but I feel that it is quite helpful, so worth exploring further.

The Rubber Ducky Method is very similar to pseudocode except that it is more often spoken than written. You can do it how it is helpful for you, but it boils down to debugging your code through talking through and describing your problem. The rubber ducky becomes the focus of your attention: an inanimate object that is happy to hear your boring problem. In essence it saves you having to describe your programming issues to a friend or colleague and enables you to get back to work without waiting to hear back from someone. Often when we encounter issues we need to talk them through, by speaking to the Rubber Ducky it becomes your confidant.

By describing a problem, we often can see it’s constituent parts, break it down, and see where we might have gone wrong.

Trying something

This is what we’ve been doing in a nutshell. Basically I have no idea what I am doing so I am constantly just trying things out and seeing what sticks.

Reading error messages

I am slowly getting better at this, better at not just seeing scary red text and actually reading them. Turns out they are quite helpful! Also, when in doubt google the error and you can guarantee someone else has encountered it.

Console.logging

This is also very very helpful. I think it will become a habit in time, much like we were learning “git status”. Doing a console.log is our one way into the code/computer to see what is going on, so it’s an important resource to have on hand.

Googling

Once again, I probably spent at least 50% of my time currently reading alternative solutions and troubleshooting errors etc by Google search.

Asking your peers for help

I think this is probably the greatest resource we all have at the moment. We are all learning and having trouble with roughly the same things at the same time, so asking for help from peers is great. Often the explanations are super helpful because they are using terminology and the understanding of someone who’s just learnt the skill. It’s also great responding because it allows you to reflect on how you solved the problem and explaining it can help solidify the learning.

Asking coaches for help

This is also very useful, especially when having difficulty grasping a concept that potentially requires a broader or more in depth understanding of the problem.

Improving your process with reflection

I think that the reflections have been helpful to process what we do each day. But importantly, without reflection we would keep repeating mistakes. So in a sense it’s the most important tool of all the one’s here, with reflection we can grow and develop.

Animals can provide a good analogy for the differences between HTML and CSS. HTML would be a turtle because it is slow and steady. Explain control flow and loops using an example process from everyday life.

Control flow in programming is the order in which function calls or statements are executed by the program. Control flow is essentially the computer program’s version of English speakers reading from left to right. For English speakers, reading left to right is our control flow. In this sentence, we read ‘In’ (1st), ‘this’ (2nd), and ‘sentence’ (3rd), that is the control flow. For humans it’s the order that the words will hit our brain.

A loop can interrupt this control flow because it may ask the function to execute an argument any number of times, so the program will run the loop for as long as you ask it to. Once the loop is completed the normal control flow will resume. You can liken this to getting stuck on a word while reading it and trying to understand it, perhaps you read over it a number of times, once you’ve understood you move on and completed the sentence.


Describe what the DOM is and an example of how you might interact with it.

What is the DOM?

It is a “Document Object Model”. In essence, it is the object representation of your entire HTML document. It is a tree like data structure, with many ‘nodes’ coming off of it much like the leaves. Everything that makes up your DOM is known as nodes. A node can be an element, attribute, text content, comments, etc.

The DOM is a hierarchical structure that your web browser uses to understand and make sense of all the different elements going on behind the scenes. Everything you use in HTML has an object representation in the DOM, which enables you to access these elements in JavaScript, and also modify them. The Document Object is not just for viewing purposes, you can actually use it to manipulate the document. If you make any changes with JavaScript they will be reflected in your browser.

The Document Object also allows for specialised interaction with the site user. Changes you make in the DOM with JavaScript can mean you can make your page respond to mouse-clicks, hovering, and detecting, when a particular key is pressed. It is essential to responsive programming. The DOM acts as they key that allows you to work with the HTML and CSS and link to the JavaScript.

What is a JavaScript ‘event’?

Events are things that happen to an HTML element. If JavaScript is incorporated into an HTML document, JavaScript can respond to events that happen with the HTML. Such as a button click, a webpage loading. JavaScript lets you execute code when an event happens – this is how we can have interactive web pages. We can add the JavaScript code to HTML elements so they can respond when the event is realised.


Explain the difference between accessing data from arrays and objects.

An array is a list of items in square brackets separated by commas, which can be used to contain data that is accessible through it’s position in the index. Strings in arrays are contained within quotes, and numbers don’t require anything. The first item in an array is at index point 0, the second at 1, and so on. Indices start at 0. Please note that arrays don’t have to be within in the square brackets, you can define the array and then provide the elements (data) that goes inside.

An object is a data type that stores information. Its structure follows key:value, in an object the information is stored in pairs. For example; food: bread, ingredient: flour, flourType: wholemeal. The “flourType” in this instance is the key, and “wholemeal” is the value. In an object a value can be a string, an array, a number, and even a function. You can access an object in one of two ways, firstly you can use objectName.propertyName, so in our previous example it would be objectName.flourType and the result would be “wholemeal”. When a property name contains spaces you need to use an array-type to retrieve the data from the object. Here we would have objectName[‘flour type’], and once again we would get “wholemeal”. Due to the confusion, if possible it’s good to avoid using spaces for the property name.


Explain what functions are and why they are useful.

A function is the building block of JavaScript, it’s where the magic happens. In essence, a function’s purpose is to allow computers to do what they do best, which is perform repetitive tasks efficiently. We can write a function to perform a loop, count up to 10,000, capitalise names in a list…the options are pretty much endless. A function is why we use computers to perform tasks and not just do them all ourselves. They are basically what computers were designed for.

Classes...ids ...who understands em?!

What are the best practices associated with using classes vs. ids?



What are classes and ids to begin with?

Classes and IDs are selectors within HTML and CSS which enable the user to choose which sections of code to apply certain rules or styling to. They make it possible to change the colour of text or change how an element is aligned on a page. Classes and IDs serve slightly different purposes which well get into next.


What is a class ?

A class is essentially a descriptor you can apply to an element to direct that element how to behave. In HTML you can write a class, and then in CSS you can define what that class will do to the element, such as change background colour of text, or create a border for an image. If you like how the class is performing on that element you can apply the same class to other elements on your page. For example, if you would like every second image to have a blue border, you can create a class "blue-border” and apply it to the images in HTML that you want to have the blue border. Then in CSS you can apply the styling to the class “blue-border”.

There is no limit on how many classes can be used in HTML, in this sense they are more flexible, but less powerful than IDs in that IDs have the ability to over-ride a class.


What is an id ?

IDs are used for a single unique element. There can only be one ID per page of HTML. IDs serve many functions that are similar to classes, with some key differences. The use of an ID in an HTML document is to point to a specific style declaration.

As we have mentioned, IDs are special and can only be used once per page. Part of their specificity is that they have precedence over classes on the page. If your ID directs an element to do one thing, such as make all text green, and then a class directs to make text red, the text will remain green because the ID takes precedence and is more powerful.

Depending on how you apply an ID to an element, you can use the ID as a function to jump to a specific section of a page. The ID can act as a pointer to a desired location. For example, I’m sure you’ve encountered a webpage where at the bottom of the page it says “click here to jump to top of page”. You can use an ID to perform this action and ones similar.
